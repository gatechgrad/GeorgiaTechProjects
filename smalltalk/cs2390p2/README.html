How do you know if a Web page is changed? You read over the page, hope that the author put some marks where something has changed, and at least put the changes right at the top or right at the bottom. Now, how you can do it algorithmically? Of course, you could save a copy of the page and do some kind of differential analysis (diff) on the two versions. But that eats up lots of space. Can you boil it down to just a couple numbers, or at least, a few characters? A length of a page is a good indicator. But how about another? Maybe...

  • Grab the first three and last three characters on a page?
  • Grab the first characters of each of the first five
  • Add up the ASCII values of the first 10 characters on a page?
  • Maybe the first 10 and the middle 10 and the last 10?
  • Maybe just count key things like the number of paragraph markers (<p>)?
  • Maybe combine several values by concatenating them (remember the comma is the string concatenation operator in Smalltalk)

You are to write a piece of code that can compute key values: a length and some kind of checksum or indicator values. Your limitation is 15 readable ASCII characters.. The length in ASCII, you can assume, will always be less than 99,999 characters. The goal is to come up a characteristic string that will change when and if the page changes.

I want to be able to use your code like this: 

HTMLPageCharacterization for: 'http://www.cnn.com'

which should return a 15 character code for that page.

Implementation Hints

See the class HTTPSocket, especially the example class methods, for how to grab a page. Your code should work for http:// and file:// URLs.

News pages are good pages to use as examples. http://www.cnn.com can change frequently in useful ways.

Scenarios

  • A class News page always has a date and a new entry at the top of the body of the page, but doesn't change the header of the page. How does your characterization deal with that?
  • The page that you reference has a large table as a navigation bar, and the frequently changing content is halfway down the page. How does your characterization deal with that?
