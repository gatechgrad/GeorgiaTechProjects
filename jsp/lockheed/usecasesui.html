Adding a new use case
1.  Choose a particular actor
2.  Enter use case title and signature at the bottom of the page
3.  Proceed to modifying a use case

Documenting, Designing, and Testing an existing use case
1.  Choose a use case for a particular actor
2.  Change or Enter information for the following categories and view existing posts:
  Purpose - the reason for the use case
  Design - discussing implementations for the use case in relation to the database setup
  Test - SQL database statments to perform the use case
3.  View Tables - check the database to ensure the use case was carried out correctly
